05

Prompt engineering: Cursor üçün ən praktik formula

Yaxşı prompt modelə nə istədiyinizi yox, həm də nəyi dəyişməməli olduğunu bildirir. Bu hissə tələbələr üçün ən vacib vərdişdir.

5 hissəli prompt formulu Role + Goal + Context + Constraints + Output
1. Rol“Sən senior frontend developersən...”
2. Məqsəd“Login səhifəsində validation bug-ını düzəlt.”
3. ContextFayl, error, mövcud struktur, API cavabı.
4. LimitlərDizaynı, public API-ni, database schema-nı pozma.
5. OutputƏvvəl plan, sonra diff, sonda test checklist.
feature-prompt.md
Sən Python müəllimi kimi kömək edirsən.
Məqsəd: kitabxana_lab.py-ə show_stats(books) funksiyası əlavə et.
Context: @kitabxana_lab.py
Limitlər:
- Yalnız standart kitabxana (json, os kifayətdirsə).
- CLI menyunu pozma; yeni seçim "3 — Statistika".
- sqlite3 cursor ilə qarışdırma — bu Cursor IDE dərsidir.
Output:
1. Əvvəl qısa plan yaz.
2. Hansı funksiyaların dəyişəcəyini göstər.
3. Minimal kod dəyişikliyi.
4. python kitabxana_lab.py test checklist.
debug-prompt.md
Bu xətanı analiz et və tələsmədən səbəbini tap:
[buraya terminal errorunu yapışdır]

Səndən istədiyim:
- Errorun kök səbəbini izah et.
- Hansı faylda problem ola biləcəyini de.
- Mənə 2 mümkün həll yolu ver.
- Ən təhlükəsiz həlli seç və niyə seçdiyini izah et.
- Kod yazmazdan əvvəl məndən təsdiq istəmə, amma dəyişiklikləri minimal saxla.

Rules, Docs və MCP: Cursor-u komanda standartına uyğunlaşdırmaq

Professional komandada AI təsadüfi işləməməlidir. Kod stili, naming convention, test qaydası və təhlükəsizlik limitləri əvvəlcədən yazılmalıdır.

.cursor/rules.md
# Project Rules

- Cavabları Azərbaycan dilində izah et, amma kod commentləri ingiliscə saxla.
- Python 3.10+; snake_case funksiya adları.
- `any` tipindən qaç; f-string ilə SQL yığma (sqlite3 üçün ? placeholder).
- Yeni feature üçün ən azı 1 test nümunəsi təklif et.
- Security: token, password, secret və env dəyərlərini koda yazma.
- Böyük dəyişiklikdə əvvəl plan ver, sonra kod yaz.

MCP nə üçün lazımdır?

MCP serverləri Cursor-a xarici alətlərlə təhlükəsiz işləmək imkanı verə bilər: dokumentasiya, issue tracker, database schema, dizayn sistemi və s. Amma tələbələrə ilk dərsdə əsas prinsip öyrədilməlidir: AI-ya yalnız lazım olan, icazəli və təhlükəsiz context verilir.

09

Risklər: AI köməkçidir, məsul developer sizsiniz

Cursor sürət verir, amma məsuliyyəti götürmür. Tələbələr bilməlidir: AI output-u həmişə review, test və security yoxlamasından keçməlidir.

!

Hallucination

AI mövcud olmayan funksiya, library və ya config uydura bilər.

!

Over-engineering

Sadə bug üçün böyük arxitektura dəyişməsi təklif edə bilər.

!

Security

Secret, token və user data paylaşımı ciddi risk yaradır.

!

Blind apply

Diff oxunmadan tətbiq edilən kod production probleminə çevrilə bilər.

Qızıl qayda

AI kod yazırsa, developer sübut tələb etməlidir: “Niyə belə etdin?”, “Hansı test bunu təsdiqləyir?”, “Hansı edge case qalıb?”
