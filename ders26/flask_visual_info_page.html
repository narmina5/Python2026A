01

Flask nədir və nə üçün istifadə olunur?

Flask Python ilə web application və API qurmaq üçün yüngül framework-dür. O, routing, request/response, template rendering və extension ekosistemini verir, amma tətbiqin strukturunu zorla diktə etmir.

minimal core

Kiçik başlayır

Bir `app.py` faylı ilə işlək web server yaratmaq mümkündür. Sonra ehtiyac olduqca `Blueprint`, database və authentication əlavə edilir.

Python functions

Route = funksiya

Hər URL adətən bir Python funksiyasına bağlanır. Funksiya string, HTML template, JSON və ya redirect qaytara bilər.

backend

Məntiq server-dədir

Database sorğuları, form validation, session və API qaydaları server tərəfində icra olunur.

02

Request lifecycle: brauzerdən Python-a, Python-dan HTML-ə

Flask-in əsas ideyası budur: URL gəlir, uyğun route seçilir, view function işləyir, nəticə response obyektinə çevrilir və browser-ə göndərilir.

HTTP request axını browser → server → browser
1. URL

İstifadəçi `/dashboard` açır və browser HTTP request yaradır.

2. WSGI server

Request Flask application obyektinə ötürülür.

3. Routing

Flask URL-i `@app.route` qaydaları ilə müqayisə edir.

4. View function

Python kodu data hazırlayır, template render edir və ya JSON yaradır.

5. Response

HTML, JSON, file və ya redirect status code ilə geri dönür.

03

Route sistemi: URL-i Python funksiyasına bağlamaq

`@app.route()` decorator kimi işləyir: aşağıdakı funksiyanı seçilmiş URL üçün cavabdeh edir. Bu, Flask-in ən vacib mexanizmidir.

app.py 
from flask import Flask, render_template, jsonify

app = Flask(__name__)

# URL: http://127.0.0.1:5000/
@app.route("/")
def home():
    return "Salam, Flask!"

# URL parameter: /users/42
@app.route("/users/<int:user_id>")
def user_detail(user_id):
    return render_template("user.html", user_id=user_id)

# API endpoint
@app.route("/api/health")
def health():
    return jsonify({"status": "ok"})
Route mapping URL → function
GET / home()
GET /users/<int:user_id> user_detail()
POST /login login()
PUT /api/products/<id> update_product()

06

Form və JSON: istifadəçi datası Flask-ə necə gəlir?

HTML form adətən `request.form` ilə, JSON API body isə `request.get_json()` ilə oxunur. Flask burada request obyektini hazır formada verir.

POST form 
from flask import request, redirect

@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        email = request.form.get("email")
        password = request.form.get("password")
        # validate user here
        return redirect("/dashboard")

    return render_template("login.html")
JSON API 
from flask import request, jsonify

@app.route("/api/products", methods=["POST"])
def create_product():
    data = request.get_json()

    product = {
        "name": data["name"],
        "price": data["price"]
    }

    return jsonify(product), 201

07

Response və status code: browser nə baş verdiyini necə anlayır?

Flask sadə string qaytarsanız belə onu HTTP response-a çevirir. Status code browser-ə və API client-ə nəticənin mənasını bildirir.

Ən çox görülən HTTP status code-lar response signals
200

OK: request uğurlu oldu.

201

Created: yeni resource yaradıldı.

302

Redirect: başqa URL-ə yönləndir.

404

Not Found: route və ya resource tapılmadı.

custom status 
@app.route("/api/products/<int:product_id>")
def get_product(product_id):
    product = find_product(product_id)

    if product is None:
        return {"error": "Product not found"}, 404

    return product, 200

Development server və deployment fərqi

`flask run` öyrənmək və local development üçündür. Production mühitində adətən Gunicorn/uWSGI kimi WSGI server və Nginx kimi reverse proxy istifadə olunur.

Production arxitekturası real server flow
User browser

HTTPS request göndərir.

Nginx

Static files verir və request-i WSGI server-ə yönləndirir.

Gunicorn

Bir neçə worker ilə Flask app-i işlədir.

Flask app

Route, business logic və template/API response yaradır.

Database

PostgreSQL, SQLite, MySQL və s. ilə data saxlanılır.

local development

Öyrənmək üçün

`debug=True`, avtomatik reload, sadə `flask run`, localhost testing. Sürətli iterasiya üçündür.

production

İstifadəçilər üçün

Environment variables, secret management, logging, WSGI server, HTTPS və monitoring lazımdır.

09

Yeni başlayanlarda ən çox qarışan yerlər

Flask sadə görünsə də, bəzi konseptlər tez-tez qarışır. Aşağıdakı qaydalar debugging vaxtına qənaət edir.

!

`templates/` və `static/` adları xüsusi məna daşıyır

Flask default olaraq bu qovluq adlarını axtarır. Adları dəyişirsinizsə, bunu app yaradılarkən göstərmək lazımdır.

!

GET və POST eyni şey deyil

GET data oxumaq, POST isə data göndərmək və ya yaratmaq üçün istifadə olunur. Form submit çox vaxt POST olmalıdır.

!

`debug=True` production-da saxlanmamalıdır

Debug mode detallı error məlumatları göstərə bilər. Bu məlumatlar real istifadəçi mühitində risklidir.

!

Secret key kodun içində hardcode edilməməlidir

Session və CSRF kimi mexanizmlər üçün secret-lər environment variable ilə idarə olunmalıdır.
